# Abstract｜玉山與 Google Cloud 合作推出 AI Agents（AI 房貸顧問、投資 i-chat）

來源：
- https://blog.google/intl/zh-tw/products/cloud/esun-bank-collaborates-with-google-cloud-on-ai-agents/
日期：2026-03-11

## 一句話摘要
玉山銀行基於 Google Cloud Vertex AI 與 Gemini 建置兩個金融 AI 代理（AI 房貸顧問、投資 i-chat），以 24/7 互動諮詢、RAG+Grounding 機制與合規安全架構，提升客戶體驗、流程效率與可衡量業務轉換。

## 文章重點（Abstract）

1. **兩個核心應用上線**
- **AI 房貸顧問**：涵蓋申貸前諮詢、申辦協助、貸後管理。
- **投資 i-chat**：把投研內容與即時資訊轉成可互動理財對話。

2. **技術底座：Vertex AI + Gemini + RAG**
- 以 Vertex AI 做企業級模型開發與部署。
- 以 Gemini 作為生成與理解引擎。
- 透過 RAG（搭配 Grounding）把回覆綁定在內部知識與外部即時資訊，強化正確性與可追溯性。

3. **房貸場景的業務價值**
- 提供 24 小時房貸諮詢與流程引導。
- 對既有客戶可大幅簡化申請步驟（文中提及約減少 90% 必填欄位）。
- 目標是降低流程摩擦、減少流失、提升申貸意願。

4. **財富管理場景的業務價值**
- 將高密度投研報告與新聞資訊即時轉成可對話內容。
- 同時服務新手與進階投資者，縮短資訊理解與決策路徑。
- 可引導至申購流程，形成「資訊→理解→行動」閉環。

5. **安全與合規主張**
- 強調資料主權（客戶資料不拿去訓練 Google 通用模型）。
- 採企業級安全基礎設施與治理機制。
- 使用 Model Armor、AlloyDB for PostgreSQL 等元件支撐安全與合規要求。

6. **產業意義**
- 展示金融業 AI agent 的可落地模式：
  - 前台：客戶互動與個人化服務
  - 中台：知識檢索與合規回覆
  - 後台：流程效率與人力槓桿
- 顯示「生成式 AI + 金融合規」可共存，且能轉化成實際商業成果。

## 可借鏡的實作框架（給你做專案參考）
- Use case 分層：房貸（流程導向）/理財（知識導向）
- 技術分層：LLM（Gemini）+ RAG + Grounding + Guardrails
- 指標分層：
  - 產品指標：轉換率、完成率、留存
  - 模型指標：正確性、幻覺率、可引用性
  - 風控指標：合規觸發率、敏感回覆攔截率

## TL;DR
這篇案例的關鍵不是「用了 LLM」，而是把 LLM 變成 **可管控、可擴展、可轉換業績** 的金融 AI 代理系統。
